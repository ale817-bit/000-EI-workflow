# graphic-system-builder

## 使用场景
- 视觉方向已确认，需要形成生产级系统规范。

## 输入材料
- 平面视觉策略文档。
- 品牌资产与落地约束。

## 工作步骤
1. 定义基础规范（grid、字体、色彩、图形）。
2. 建立高复用模块与组合规则。
3. 绑定应用场景（展墙、导视、展板等）。
4. 明确输出格式、命名与交付约定。

## 输出格式
- 参考 `templates/graphic-system-template.md`。
- 规范应可直接用于设计执行与审校。

## 检查标准
- 规则完整且不存在冲突。
- 模块可复用、可拓展。
- 交付规范对制作方友好。

## 禁止事项
- 不得仅输出概念，不提供落地规则。
- 不得忽略命名和文件组织规范。
