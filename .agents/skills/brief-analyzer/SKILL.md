# brief-analyzer

## 使用场景
- 项目启动阶段，需要将复杂 brief 拆解为可执行需求。

## 输入材料
- `input/briefs/` 中的策划案与需求文档。
- `input/brand-assets/` 中的品牌约束信息（如有）。

## 工作步骤
1. 识别项目目标、受众、场景与约束。
2. 提取显性需求与隐性需求。
3. 将需求分层为必须项/可选项/风险项。
4. 输出待确认问题列表并排序优先级。

## 输出格式
- 参考 `templates/requirement-analysis-template.md`。
- 建议输出为 Markdown，包含结论摘要与待确认项。

## 检查标准
- 需求项可追溯到输入来源。
- 需求拆解具备优先级。
- 风险与依赖已明确。

## 禁止事项
- 不得编造输入材料中不存在的业务目标。
- 不得跳过待确认项直接给出最终定案。
