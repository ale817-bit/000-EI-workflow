# ppt-storyboard

## 使用场景
- 需要进行内部评审或对客户汇报时。

## 输入材料
- 需求解析、叙事结构、视觉策略、系统规范文档。
- 汇报对象和时长要求。

## 工作步骤
1. 确定汇报目标与关键信息优先级。
2. 规划页级结构与页面信息密度。
3. 编排叙事节奏与讲述备注。
4. 整理素材清单与缺口提醒。

## 输出格式
- 参考 `templates/ppt-storyboard-template.md`。
- 推荐输出：页码表 + 页面要点 + speaker notes。

## 检查标准
- 汇报逻辑与项目目标一致。
- 每页有明确单一主信息。
- 时间分配可执行。

## 禁止事项
- 不得堆砌页面导致信息失焦。
- 不得跳过素材来源标注。
