# exhibition-narrative

## 使用场景
- 已完成需求解析，需要构建叙事结构与内容动线。

## 输入材料
- 需求解析文档。
- `input/references/` 中的案例与内容参考。

## 工作步骤
1. 定义叙事命题与核心信息。
2. 设计主线/分线与信息层级。
3. 建立空间区域与内容映射。
4. 输出叙事节奏（开场-展开-高潮-收束）。

## 输出格式
- 参考 `templates/narrative-structure-template.md`。
- 使用表格描述区域与内容关系。

## 检查标准
- 叙事逻辑闭环，且目标受众可理解。
- 区域映射与信息层级一致。
- 对下一阶段视觉策略有明确输入。

## 禁止事项
- 不得脱离需求文档单独构建叙事。
- 不得使用空泛口号替代结构化内容。
