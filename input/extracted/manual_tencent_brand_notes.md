# 腾讯品牌视觉约束说明｜Tencent Brand Notes for Exhibition Graphic System 

## 0. 文件用途

本文是根据腾讯品牌规范文件与腾讯 WAIC 视觉简报整理出的品牌视觉约束说明，用于 000-EI-workflow 后续阶段：

- visual-strategy
- graphic-system-builder
- ppt-storyboard
- design-review

本文不是完整 VI 手册替代品，而是为展厅 / 展位 / 临展平面视觉系统生成提供可执行的品牌约束、视觉转译规则和禁止事项。

后续 Codex 在生成视觉策略和平面视觉系统时，必须优先遵守本文，不得与本文规则冲突。

## 1. 品牌基础标志规则

### 1.1 品牌基础标志构成

腾讯品牌基础标志由英文字标与中文字标组成。其元素之间的相对大小和位置是固定的，不应重新绘制、重新组合或自行调整比例。

### 1.2 使用规则

- 英文字标 Tencent 可以独立使用。
- 中文字标“腾讯”不可单独使用。
- 腾讯标志应从最终规范文件中直接拷贝，不得重新绘制。
- 不得擅自组合腾讯标志与其他图形、基础符号或临时图案。
- 对外宣传、品牌推广、企业形象展示中，应优先使用品牌基础标志。

### 1.3 禁止事项

不得出现以下情况：

- 将基础符号与腾讯标准字错误组合。
- 将基础符号与错误的标志组合。
- 随意改变基础符号的局部形状。
- 将中文字标单独作为正式品牌标志使用。
- 重新绘制 Tencent / 腾讯字标。
- 在展厅主视觉中把基础符号当成主图形标志使用。

## 2. 基础符号规则

### 2.1 基础符号来源

腾讯基础符号来源于品牌基础标志中的局部元素。基础符号可以作为辅助图形或局部图形使用，但不能作为主图形标志使用。

### 2.2 基础符号 1

适用场景：

- 可作为纹样图形使用。
- 可作为页面角标使用。
- 除黑白场景外，建议使用腾讯蓝或弱化色。

不适合：

- 不能作为主视觉核心符号。
- 不能替代 Tencent / 腾讯品牌标志。
- 不能与标准字擅自组合成新标志。

### 2.3 基础符号 2 与基础符号 3

适用场景：

- 可作为局部底图使用。
- 可作为标志系统中的辅助元素。
- 可结合字标进行局部设计。
- 可结合图标进行局部设计。

注意事项：

- 使用时必须适当调整透明度、比例和位置。
- 不得喧宾夺主。
- 不得破坏腾讯品牌识别一致性。
- 不得被误认为新的品牌 Logo。

## 3. 字体规范

### 3.1 标题字体

腾讯体 W7 可作为标题字使用。

使用规则：

- 腾讯体 W7 适合用于标题、强调信息、关键数字或重点语句。
- 腾讯体 W7 不推荐大面积使用。
- 标题中使用腾讯体 W7 时，应保持清晰、有力、简洁。

### 3.2 正文字体

正文优先使用：

1. 思源黑体
2. 腾讯体 W3

使用规则：

- 正文信息较多时，优先使用思源黑体。
- 腾讯体 W3 可用于辅助信息、说明文字或与 W7 搭配使用。
- 腾讯体 W7 + 腾讯体 W3 可以形成标题与正文层级，但 W7 只能承担强调作用。

### 3.3 中英文字体关系

使用腾讯体中文时，应对应使用腾讯体英文。

后续在展厅平面视觉系统中，若出现中英文混排，应保持中英文风格一致，不应出现明显割裂。

## 4. 延展图形系统核心规则

### 4.1 核心理念

腾讯延展图形系统是一个可变动的完整视觉体系。其基础元素遵循“斜 8 度切割”的原则，可根据实际应用情况调整图形长度、高度、颜色和切割比例。

后续所有展厅平面视觉延展，都应围绕“斜 8 度”建立视觉秩序，而不是随意使用科技线条或装饰噪声。

### 4.2 斜 8 度原则

斜 8 度是腾讯延展图形的底层结构规则。

可用于：

- 版式切割
- 背景结构
- 信息卡片边界
- 展板标题区
- 屏幕 UI 模块
- 导视牌面图形结构
- 展厅墙面平面图形
- 动态视觉中的转场方向

禁止：

- 使用与斜 8 度冲突的随机斜线系统。
- 使用过度赛博朋克化的霓虹线条替代斜 8 度结构。
- 仅把斜线当成装饰，而不形成版式秩序。
- 让斜 8 度图形压过品牌信息和展项内容。

## 5. 斜线网格系统

### 5.1 网格结构

腾讯延展图形包含：

- 基础网格
- 斜线辅助网格
- 纵向网格
- 横向网格

斜线网格应与页面比例、文字排版和信息模块形成整体关系。

### 5.2 在展厅平面中的应用

可用于：

- 展板版式骨架
- 大标题区背景
- 信息图分区
- 展项说明牌
- 导视底纹
- 多媒体屏幕 UI 背景
- 动线图、系统架构图、产业链图等信息图

使用时应注意：

- 网格可见度应根据场景调整。
- 信息阅读优先于装饰效果。
- 背景网格不可干扰正文。
- 大面积应用时需要降低对比度。
- 局部强调时可提高蓝色层级或透明度。

## 6. 延展图形系统

### 6.1 延展图形基础样式

延展图形由蓝色矩形块、斜 8 度切割、水平层次和透明度变化组成。

图形可根据应用场景调整：

- 长度
- 高度
- 颜色层级
- 切割比例
- 透明度
- 图形疏密
- 图层数量

### 6.2 样式层次

延展图形可分为由“稳定”到“灵活”的不同层级：

1. 稳定结构
2. 三等分结构
3. 多格结构
4. 横向结构
5. 灵活结构

设计判断：

- 严肃信息、品牌规范、导视系统应使用更稳定的结构。
- 活动传播、互动区域、AI 体验区可使用更灵活的结构。
- 技术内容、产业方案、To B 信息应避免过度活泼。
- C 端互动、AI 伙伴、生活化场景可适度增加图形灵活度。

## 7. 样式 A 视觉规则

### 7.1 样式 A 的特征

样式 A 更强调从远到近、从整体到局部的视觉层次，适合建立展厅整体主视觉和较强识别感。

可用于：

- 主视觉 KV
- 主入口背景
- 大型展板
- 核心数据展示
- 主标题区
- 主题墙
- 重要屏幕首页

### 7.2 样式 A 的视觉控制

样式 A 可在以下维度变化：

- 信息密度：疏到密
- 图形节奏：静态到动态
- 图层深度：浅层到深层
- 背景占比：弱背景到强背景

使用建议：

- 主入口应使用较强识别度，但不能影响品牌标志。
- 数据展示可结合大数字、横向分割和蓝色层级。
- 互动区域可适度增加图形节奏，但要保持秩序。
- 不应让背景图形过度复杂，影响展项内容阅读。

---

## 8. 样式 B 视觉规则

### 8.1 样式 B 的特征

样式 B 更适合应用于包装、局部物料、辅助视觉和多场景延展。它强调横向层次、透明色块、稳定版式和局部灵活变化。

可用于：

- 延展物料
- 手提袋
- 卡片
- 邀请函
- 礼品包装
- 小型展项说明牌
- 周边物料
- 辅助视觉系统

### 8.2 样式 B 的使用原则

- 可根据实际内容调整图形层数。
- 应保持横向节奏和斜 8 度切割关系。
- 可使用灰度、蓝色、紫色等不同色彩体系。
- 用于物料时，应保持品牌标志清晰可读。
- 不应将样式 B 直接放大为过度复杂的展厅主视觉。

## 9. 色彩系统

### 9.1 主色

腾讯蓝是本次视觉系统的唯一核心主色。

色值：

- Tencent Blue
- RGB: 0 / 82 / 217
- CMYK: C100 M60 Y0 K0
- HEX: #0052D9

使用规则：

- 腾讯蓝应作为全案视觉骨架。
- 主标题、关键信息、品牌识别区、主背景结构应优先使用腾讯蓝。
- 不应随意替换为其他蓝色。
- 不应让辅助色超过腾讯蓝的品牌主导地位。

### 9.2 辅助色

从 WAIC 视觉简报中可读取到的辅助色包括：

- 青色：RGB 0 / 169 / 206
- 绿色：RGB 121 / 192 / 0
- 紫色：RGB 155 / 38 / 182

使用建议：

- 青色可用于 AI、数据、连接、响应、轻盈科技感。
- 绿色可用于生态、开放、共创、可持续、伙伴关系。
- 紫色可用于智能、创意、未来感、情绪化互动。
- 辅助色应作为局部强调色，不可替代腾讯蓝成为主色。
- 大面积辅助色使用需谨慎，避免偏离腾讯品牌识别。

### 9.3 渐变与柔和过渡

WAIC 视觉方向中使用柔和渐变承接“AI 友邻”的陪伴属性，表达温柔回应、双向对话和平等松弛感。

使用建议：

- 渐变应基于腾讯蓝展开。
- 可使用蓝到青、蓝到紫、蓝到浅蓝的过渡。
- 避免高饱和霓虹、重赛博朋克、过度炫光。
- 渐变应服务信息层级，而不是单纯装饰。

## 10. WAIC 视觉简报核心方向

### 10.1 核心概念

WAIC 视觉简报提出的核心概念是：

AI 友邻

其含义是：

- 将 AI 从遥不可及的科技奇观转译为身边的友邻。
- 不再强调冰冷、单向输出、工具化的 AI。
- 强调平等陪伴、双向对话、靠谱兜底、全场景适配。
- 使用“对话”作为核心视觉符号。
- 在遵守腾讯斜 8 度规范的基础上，表达更松弛、更亲和、更有互动感的 AI 关系。

### 10.2 “友”的属性

“友”代表：

- 鲜活
- 有情绪
- 有互动感
- 有亲和力
- 有“活人感”
- 适合 C 端生活场景
- 适合互动传播

视觉转译方式：

- 更柔和的色彩过渡
- 对话气泡或双向交流符号
- 轻量图形节奏
- 圆角或更具亲和力的信息模块
- 更低压的信息呈现方式
- 适度增加情绪化辅助色

### 10.3 “邻”的属性

“邻”代表：

- 稳重
- 有序
- 专业
- 可靠
- 有承载力
- 适合 To B 产业基建
- 适合专业内容输出

视觉转译方式：

- 稳定版式结构
- 明确网格系统
- 克制的信息层级
- 低噪声背景
- 可信赖的品牌蓝主导
- 适合展示产业方案、技术能力、行业案例

### 10.4 “AI 友邻”的视觉平衡

后续视觉设计不得只做“可爱化 AI”，也不得只做“冰冷科技 AI”。

必须同时满足：

- 有亲和力，但不幼稚。
- 有科技感，但不冰冷。
- 有互动感，但不混乱。
- 有品牌统一性，但不僵硬。
- 能服务 C 端传播，也能承载 To B 专业内容。

## 11. 标题设计方向

### 11.1 主标题造型

WAIC 视觉简报中，主标题以“对话气泡”为核心造型，呼应“对话”主题和“AI 友邻”定位。

### 11.2 标题设计规则

- 必须遵循腾讯斜 8 度规范。
- 可使用腾讯蓝渐变。
- 兼顾品牌专业度与陪伴温度感。
- 不应使用与腾讯品牌冲突的独立字体风格。
- 不应过度手写、过度卡通或过度赛博。

### 11.3 展厅平面应用

可用于：

- 主题墙标题
- 分区标题
- 展板一级标题
- 屏幕首页标题
- 宣发 KV 标题
- 导览页主标题

## 12. 展厅平面视觉系统转译规则

### 12.1 应用对象

后续 graphic-system-builder 阶段应将以上品牌规则转译到：

- 主视觉 KV
- 展板系统
- 导视 / 标识系统
- 分区标题系统
- 信息图系统
- 屏幕 UI / 多媒体画面
- 展项说明牌
- 宣发物料
- 周边物料
- PPT 汇报系统

### 12.2 展板系统

建议：

- 使用腾讯蓝作为主背景或主结构色。
- 使用斜 8 度切割建立标题区和内容区关系。
- 大标题可使用腾讯体 W7 或对应标题规范。
- 正文优先使用思源黑体。
- 信息较多时使用模块化卡片，不要堆叠大段文字。
- 技术内容应以信息图、流程图、矩阵图、能力图谱方式表达。

### 12.3 导视 / 标识系统

建议：

- 以腾讯蓝和白色为主。
- 使用斜 8 度边界或局部切割增强品牌识别。
- 避免背景网格过密影响识别。
- 导视信息必须高对比、远距离可读。
- 基础符号只能作为局部辅助，不可作为主导视标志。

### 12.4 信息图系统

建议：

- 使用斜 8 度网格统一信息图结构。
- 数据、能力、行业方案、产品关系应采用模块化表达。
- 可使用蓝色层级区分主次。
- 青色、绿色、紫色仅用于局部强调类别。
- 避免复杂发光线条和无意义数据粒子。

### 12.5 屏幕 UI / 多媒体界面

建议：

- 保持腾讯蓝基底。
- 采用卡片化、模块化、对话式信息布局。
- AI 互动界面可加入对话气泡、双向箭头、连接节点等元素。
- 不应让 UI 过度游戏化或赛博朋克化。
- To B 内容应保持清晰、可信、有秩序。

### 12.6 宣发物料

建议：

- 可使用更灵活的样式 A 或样式 B。
- 可适度增强辅助色、渐变、对话图形。
- 仍需保持腾讯蓝主导。
- 标题系统和图形系统应与展厅一致。

## 14. 禁止和风险提示

### 14.1 不建议使用

- 过度赛博朋克
- 高噪声数据粒子
- 复杂霓虹线条
- 过多发光效果
- 冰冷黑底蓝光风格
- 与腾讯斜 8 度无关的随机几何
- 大量非腾讯品牌色作为主视觉
- 卡通化过重的 AI 形象
- 过于幼稚的插画风格
- 把基础符号误用成主 Logo

### 14.2 必须人工复核

以下内容需要设计师人工复核：

- 腾讯品牌 Logo 使用方式
- 基础符号的实际矢量文件
- 腾讯体授权和字体文件
- PDF 中未完整解析的详细规范
- PNG 图片中的视觉信息
- 真实展厅物料尺寸与应用比例
- 是否存在甲方额外品牌限制

## 1. 品牌基础标志规则

### 1.1 品牌基础标志构成

腾讯品牌基础标志由英文字标与中文字标组成。其元素之间的相对大小和位置是固定的，不应重新绘制、重新组合或自行调整比例。

### 1.2 使用规则

- 英文字标 Tencent 可以独立使用。
- 中文字标“腾讯”不可单独使用。
- 腾讯标志应从最终规范文件中直接拷贝，不得重新绘制。
- 不得擅自组合腾讯标志与其他图形、基础符号或临时图案。
- 对外宣传、品牌推广、企业形象展示中，应优先使用品牌基础标志。

### 1.3 禁止事项

不得出现以下情况：

- 将基础符号与腾讯标准字错误组合。
- 将基础符号与错误的标志组合。
- 随意改变基础符号的局部形状。
- 将中文字标单独作为正式品牌标志使用。
- 重新绘制 Tencent / 腾讯字标。
- 在展厅主视觉中把基础符号当成主图形标志使用。

## 2. 基础符号规则

### 2.1 基础符号来源

腾讯基础符号来源于品牌基础标志中的局部元素。基础符号可以作为辅助图形或局部图形使用，但不能作为主图形标志使用。

### 2.2 基础符号 1

适用场景：

- 可作为纹样图形使用。
- 可作为页面角标使用。
- 除黑白场景外，建议使用腾讯蓝或弱化色。

不适合：

- 不能作为主视觉核心符号。
- 不能替代 Tencent / 腾讯品牌标志。
- 不能与标准字擅自组合成新标志。

### 2.3 基础符号 2 与基础符号 3

适用场景：

- 可作为局部底图使用。
- 可作为标志系统中的辅助元素。
- 可结合字标进行局部设计。
- 可结合图标进行局部设计。

注意事项：

- 使用时必须适当调整透明度、比例和位置。
- 不得喧宾夺主。
- 不得破坏腾讯品牌识别一致性。
- 不得被误认为新的品牌 Logo。

## 3. 字体规范

### 3.1 标题字体

腾讯体 W7 可作为标题字使用。

使用规则：

- 腾讯体 W7 适合用于标题、强调信息、关键数字或重点语句。
- 腾讯体 W7 不推荐大面积使用。
- 标题中使用腾讯体 W7 时，应保持清晰、有力、简洁。

### 3.2 正文字体

正文优先使用：

1. 思源黑体
2. 腾讯体 W3

使用规则：

- 正文信息较多时，优先使用思源黑体。
- 腾讯体 W3 可用于辅助信息、说明文字或与 W7 搭配使用。
- 腾讯体 W7 + 腾讯体 W3 可以形成标题与正文层级，但 W7 只能承担强调作用。

### 3.3 中英文字体关系

使用腾讯体中文时，应对应使用腾讯体英文。

后续在展厅平面视觉系统中，若出现中英文混排，应保持中英文风格一致，不应出现明显割裂。

## 4. 延展图形系统核心规则

### 4.1 核心理念

腾讯延展图形系统是一个可变动的完整视觉体系。其基础元素遵循“斜 8 度切割”的原则，可根据实际应用情况调整图形长度、高度、颜色和切割比例。

后续所有展厅平面视觉延展，都应围绕“斜 8 度”建立视觉秩序，而不是随意使用科技线条或装饰噪声。

### 4.2 斜 8 度原则

斜 8 度是腾讯延展图形的底层结构规则。

可用于：

- 版式切割
- 背景结构
- 信息卡片边界
- 展板标题区
- 屏幕 UI 模块
- 导视牌面图形结构
- 展厅墙面平面图形
- 动态视觉中的转场方向

禁止：

- 使用与斜 8 度冲突的随机斜线系统。
- 使用过度赛博朋克化的霓虹线条替代斜 8 度结构。
- 仅把斜线当成装饰，而不形成版式秩序。
- 让斜 8 度图形压过品牌信息和展项内容。

## 5. 斜线网格系统

### 5.1 网格结构

腾讯延展图形包含：

- 基础网格
- 斜线辅助网格
- 纵向网格
- 横向网格

斜线网格应与页面比例、文字排版和信息模块形成整体关系。

### 5.2 在展厅平面中的应用

可用于：

- 展板版式骨架
- 大标题区背景
- 信息图分区
- 展项说明牌
- 导视底纹
- 多媒体屏幕 UI 背景
- 动线图、系统架构图、产业链图等信息图

使用时应注意：

- 网格可见度应根据场景调整。
- 信息阅读优先于装饰效果。
- 背景网格不可干扰正文。
- 大面积应用时需要降低对比度。
- 局部强调时可提高蓝色层级或透明度。

## 6. 延展图形系统

### 6.1 延展图形基础样式

延展图形由蓝色矩形块、斜 8 度切割、水平层次和透明度变化组成。

图形可根据应用场景调整：

- 长度
- 高度
- 颜色层级
- 切割比例
- 透明度
- 图形疏密
- 图层数量

### 6.2 样式层次

延展图形可分为由“稳定”到“灵活”的不同层级：

1. 稳定结构
2. 三等分结构
3. 多格结构
4. 横向结构
5. 灵活结构

设计判断：

- 严肃信息、品牌规范、导视系统应使用更稳定的结构。
- 活动传播、互动区域、AI 体验区可使用更灵活的结构。
- 技术内容、产业方案、To B 信息应避免过度活泼。
- C 端互动、AI 伙伴、生活化场景可适度增加图形灵活度。

## 7. 样式 A 视觉规则

### 7.1 样式 A 的特征

样式 A 更强调从远到近、从整体到局部的视觉层次，适合建立展厅整体主视觉和较强识别感。

可用于：

- 主视觉 KV
- 主入口背景
- 大型展板
- 核心数据展示
- 主标题区
- 主题墙
- 重要屏幕首页

### 7.2 样式 A 的视觉控制

样式 A 可在以下维度变化：

- 信息密度：疏到密
- 图形节奏：静态到动态
- 图层深度：浅层到深层
- 背景占比：弱背景到强背景

使用建议：

- 主入口应使用较强识别度，但不能影响品牌标志。
- 数据展示可结合大数字、横向分割和蓝色层级。
- 互动区域可适度增加图形节奏，但要保持秩序。
- 不应让背景图形过度复杂，影响展项内容阅读。

---

## 8. 样式 B 视觉规则

### 8.1 样式 B 的特征

样式 B 更适合应用于包装、局部物料、辅助视觉和多场景延展。它强调横向层次、透明色块、稳定版式和局部灵活变化。

可用于：

- 延展物料
- 手提袋
- 卡片
- 邀请函
- 礼品包装
- 小型展项说明牌
- 周边物料
- 辅助视觉系统

### 8.2 样式 B 的使用原则

- 可根据实际内容调整图形层数。
- 应保持横向节奏和斜 8 度切割关系。
- 可使用灰度、蓝色、紫色等不同色彩体系。
- 用于物料时，应保持品牌标志清晰可读。
- 不应将样式 B 直接放大为过度复杂的展厅主视觉。

## 9. 色彩系统

### 9.1 主色

腾讯蓝是本次视觉系统的唯一核心主色。

色值：

- Tencent Blue
- RGB: 0 / 82 / 217
- CMYK: C100 M60 Y0 K0
- HEX: #0052D9

使用规则：

- 腾讯蓝应作为全案视觉骨架。
- 主标题、关键信息、品牌识别区、主背景结构应优先使用腾讯蓝。
- 不应随意替换为其他蓝色。
- 不应让辅助色超过腾讯蓝的品牌主导地位。

### 9.2 辅助色

从 WAIC 视觉简报中可读取到的辅助色包括：

- 青色：RGB 0 / 169 / 206
- 绿色：RGB 121 / 192 / 0
- 紫色：RGB 155 / 38 / 182

使用建议：

- 青色可用于 AI、数据、连接、响应、轻盈科技感。
- 绿色可用于生态、开放、共创、可持续、伙伴关系。
- 紫色可用于智能、创意、未来感、情绪化互动。
- 辅助色应作为局部强调色，不可替代腾讯蓝成为主色。
- 大面积辅助色使用需谨慎，避免偏离腾讯品牌识别。

### 9.3 渐变与柔和过渡

WAIC 视觉方向中使用柔和渐变承接“AI 友邻”的陪伴属性，表达温柔回应、双向对话和平等松弛感。

使用建议：

- 渐变应基于腾讯蓝展开。
- 可使用蓝到青、蓝到紫、蓝到浅蓝的过渡。
- 避免高饱和霓虹、重赛博朋克、过度炫光。
- 渐变应服务信息层级，而不是单纯装饰。

## 10. WAIC 视觉简报核心方向

### 10.1 核心概念

WAIC 视觉简报提出的核心概念是：

AI 友邻

其含义是：

- 将 AI 从遥不可及的科技奇观转译为身边的友邻。
- 不再强调冰冷、单向输出、工具化的 AI。
- 强调平等陪伴、双向对话、靠谱兜底、全场景适配。
- 使用“对话”作为核心视觉符号。
- 在遵守腾讯斜 8 度规范的基础上，表达更松弛、更亲和、更有互动感的 AI 关系。

### 10.2 “友”的属性

“友”代表：

- 鲜活
- 有情绪
- 有互动感
- 有亲和力
- 有“活人感”
- 适合 C 端生活场景
- 适合互动传播

视觉转译方式：

- 更柔和的色彩过渡
- 对话气泡或双向交流符号
- 轻量图形节奏
- 圆角或更具亲和力的信息模块
- 更低压的信息呈现方式
- 适度增加情绪化辅助色

### 10.3 “邻”的属性

“邻”代表：

- 稳重
- 有序
- 专业
- 可靠
- 有承载力
- 适合 To B 产业基建
- 适合专业内容输出

视觉转译方式：

- 稳定版式结构
- 明确网格系统
- 克制的信息层级
- 低噪声背景
- 可信赖的品牌蓝主导
- 适合展示产业方案、技术能力、行业案例

### 10.4 “AI 友邻”的视觉平衡

后续视觉设计不得只做“可爱化 AI”，也不得只做“冰冷科技 AI”。

必须同时满足：

- 有亲和力，但不幼稚。
- 有科技感，但不冰冷。
- 有互动感，但不混乱。
- 有品牌统一性，但不僵硬。
- 能服务 C 端传播，也能承载 To B 专业内容。

## 11. 标题设计方向

### 11.1 主标题造型

WAIC 视觉简报中，主标题以“对话气泡”为核心造型，呼应“对话”主题和“AI 友邻”定位。

### 11.2 标题设计规则

- 必须遵循腾讯斜 8 度规范。
- 可使用腾讯蓝渐变。
- 兼顾品牌专业度与陪伴温度感。
- 不应使用与腾讯品牌冲突的独立字体风格。
- 不应过度手写、过度卡通或过度赛博。

### 11.3 展厅平面应用

可用于：

- 主题墙标题
- 分区标题
- 展板一级标题
- 屏幕首页标题
- 宣发 KV 标题
- 导览页主标题

## 12. 展厅平面视觉系统转译规则

### 12.1 应用对象

后续 graphic-system-builder 阶段应将以上品牌规则转译到：

- 主视觉 KV
- 展板系统
- 导视 / 标识系统
- 分区标题系统
- 信息图系统
- 屏幕 UI / 多媒体画面
- 展项说明牌
- 宣发物料
- 周边物料
- PPT 汇报系统

### 12.2 展板系统

建议：

- 使用腾讯蓝作为主背景或主结构色。
- 使用斜 8 度切割建立标题区和内容区关系。
- 大标题可使用腾讯体 W7 或对应标题规范。
- 正文优先使用思源黑体。
- 信息较多时使用模块化卡片，不要堆叠大段文字。
- 技术内容应以信息图、流程图、矩阵图、能力图谱方式表达。

### 12.3 导视 / 标识系统

建议：

- 以腾讯蓝和白色为主。
- 使用斜 8 度边界或局部切割增强品牌识别。
- 避免背景网格过密影响识别。
- 导视信息必须高对比、远距离可读。
- 基础符号只能作为局部辅助，不可作为主导视标志。

### 12.4 信息图系统

建议：

- 使用斜 8 度网格统一信息图结构。
- 数据、能力、行业方案、产品关系应采用模块化表达。
- 可使用蓝色层级区分主次。
- 青色、绿色、紫色仅用于局部强调类别。
- 避免复杂发光线条和无意义数据粒子。

### 12.5 屏幕 UI / 多媒体界面

建议：

- 保持腾讯蓝基底。
- 采用卡片化、模块化、对话式信息布局。
- AI 互动界面可加入对话气泡、双向箭头、连接节点等元素。
- 不应让 UI 过度游戏化或赛博朋克化。
- To B 内容应保持清晰、可信、有秩序。

### 12.6 宣发物料

建议：

- 可使用更灵活的样式 A 或样式 B。
- 可适度增强辅助色、渐变、对话图形。
- 仍需保持腾讯蓝主导。
- 标题系统和图形系统应与展厅一致。

## 14. 禁止和风险提示

### 14.1 不建议使用

- 过度赛博朋克
- 高噪声数据粒子
- 复杂霓虹线条
- 过多发光效果
- 冰冷黑底蓝光风格
- 与腾讯斜 8 度无关的随机几何
- 大量非腾讯品牌色作为主视觉
- 卡通化过重的 AI 形象
- 过于幼稚的插画风格
- 把基础符号误用成主 Logo

### 14.2 必须人工复核

以下内容需要设计师人工复核：

- 腾讯品牌 Logo 使用方式
- 基础符号的实际矢量文件
- 腾讯体授权和字体文件
- PDF 中未完整解析的详细规范
- PNG 图片中的视觉信息
- 真实展厅物料尺寸与应用比例
- 是否存在甲方额外品牌限制
